# LLM voice on the Verdict

Status: ready-for-agent

## What to build

The `llm-voice` boundary — where "Effective + Creative AI" shows. A **server-side**
route (Claude key never in the browser) takes the already-validated decision
object from `verdict-engine`, asks Claude to render it into a human, opinionated,
confident verdict (returned as JSON), then **re-validates every number against the
computed substance before display**. The LLM owns the voice; it never originates a
number or an action (ADR-0002 / the "code owns substance" principle). Replace the
deterministic placeholder voice from `04`.

Model: `claude-opus-4-8` for verdict quality (may split a cheaper/faster model per
surface later).

## Acceptance criteria

- [ ] Verdict text is generated by Claude from the validated decision object, server-side
- [ ] A number-validation guard rejects/repairs any LLM payload whose figures disagree with the computed substance (cheap smoke test on this guard)
- [ ] The "don't fill today" bridge verdict reads as confident and counter-intuitive
- [ ] Claude API key is never exposed to the client
- [ ] If the LLM call fails, the deterministic placeholder voice still renders (never blank on stage)

## Comments

Built (AFK). `@anthropic-ai/sdk` (0.104.1) added.

- `src/ai/llm-voice.ts` (`server-only`) → `voiceVerdict(decision)` calls Claude
  **`claude-opus-4-8`** server-side (adaptive thinking, `effort: low` for a
  one-liner), then number-validates and returns the text.
- `src/ai/validate.ts` `isVoiceFaithful(text, decision)` — the guard: every RM /
  litre / day figure in the voiced line must match a verdict-engine number;
  ignores non-figures ("the 1st", "RON95"). Smoke-tested in `tests/validate.ts`
  (5 cases: accepts the fallback + a faithful rephrase, rejects an inflated
  saving and a wrong litre count).
- `src/ai/prompts.ts` — system prompt pins "use ONLY the given numbers, no
  carbon, lead with the call". `src/app/api/voice/route.ts` (POST) **recomputes**
  the decision from the scenario server-side (substance is server-authoritative)
  before voicing, so a tampered client payload can't smuggle a number in.
- `Verdict.tsx` fetches `/api/voice` (debounced) and shows the deterministic line
  until Claude's validated text arrives; any failure keeps the fallback.

"A new price posts in 2 day(s). Bridge just 4 L …" (no saving claimed). Set the
